# Security Policy

## 支持范围

当前仓库优先维护默认分支上的最新版本。

## 报告安全问题

如果你发现以下问题，建议不要直接公开提交 Issue：

- 管理权限绕过
- 会话泄露
- 密钥暴露
- 对 LiteLLM 管理接口的越权调用
- 可导致敏感配置泄露的前后端缺陷

请通过私下渠道联系维护者，或在仓库中使用非公开方式描述风险。

## 部署建议

- 不要把本项目直接暴露在公网
- 不要把 `LITELLM_MASTER_KEY` 写入仓库
- 使用独立环境变量或密钥管理系统注入配置
- 建议在反向代理、VPN 或内网环境下访问
- 如需对外使用，建议在外层补一层统一鉴权
